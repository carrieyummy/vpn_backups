# vpn_backups

用于备份和复用 VPN / Mihomo / Clash 相关配置的个人项目。核心目标是让电脑或手机能够稳定访问国外 AI 工具，并让 VPN 客户端自动选择非中国大陆节点，减少手动切换节点的成本。

## 项目目标

- 让 OpenAI、ChatGPT 等国外 AI 服务默认走可用的海外节点。
- 尽量避免 AI 流量误走中国大陆、香港、台湾等不适合的节点。
- 通过自动测速、故障转移和兜底策略，让客户端在节点异常时自动切换。
- 保留不同平台的配置和脚本，方便重装系统、迁移设备或快速恢复环境。

## 如何选择

如果你使用的是桌面或移动端 Clash 系客户端：

- 进入 [`clash_overrides/`](./clash_overrides/)。
- 按其中说明把 `Script.js` 配置为客户端的覆写脚本。
- 让客户端生成面向 AI 访问的自动选择、自动兜底策略组。

如果你使用的是 Linux 机器、软路由或远程主机：

- 进入 [`linux_mihomo/`](./linux_mihomo/)。
- 按其中说明安装 Mihomo、同步配置并启动用户级服务。
- 通过 MetaCubeXD 面板选择适合日常使用的兜底策略组。

## 使用前准备

本项目只保存配置模板、脚本和安装辅助文件，不提供代理节点本身。你需要先准备自己的订阅链接或节点信息。

建议把真实订阅链接只写入本机实际运行目录中的配置文件，不要提交到 Git 仓库。公开仓库中应保留占位符或脱敏配置，避免泄露账号、订阅地址和流量信息。

## 典型工作流

1. 克隆或复制本项目到目标设备。
2. 根据使用场景进入对应子目录。
3. 阅读对应 README，并按步骤安装或导入配置。
4. 在客户端面板中选择 AI 访问相关的自动兜底策略组。
5. 打开 ChatGPT、OpenAI API 或其他国外 AI 工具验证访问是否正常。

## 注意事项

- 节点命名会影响自动分组效果。订阅中的节点名称最好包含国家或地区关键词。脚本已经尽量适配所有的关键词，如有遗漏，请用 AI 工具自行修改。
- 自动选择不等于永久固定节点，客户端会根据测速和可用性切换。
- 如果某些国内服务、Microsoft 服务或办公软件异常，优先检查直连规则和 DNS 配置。
- 修改配置后通常需要重新加载订阅、重启 Mihomo，或在客户端中重新应用配置。
